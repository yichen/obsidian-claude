## Infrastructure Migration

### Initial Setup (AWS)
- Data ingestion to S3 object store
- Kafka
- Redshift for analytics
- EKS (Elastic Kubernetes Service)

### Migration to On-Prem (9 months ago)
- Moved from AWS to rented data center space
- Replaced Redshift with on-prem solution
- Standardized on S3-compatible object store
- Kubernetes for compute
- Iceberg table format
- Trino as processing engine

### Why On-Prem Instead of Cloud?
**Primary reason:** S3 egress charges were too expensive when sharing data with investors (Ford, VW)
- Founders' previous company experience with high egress costs
- Decision made from the start to be on-prem
