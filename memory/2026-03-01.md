- Updated `Finance/CLAUDE.md` with CMA transaction docs, backup/recovery section, and full rebuild procedure

**What was learned**:
- Fidelity CMA Activity sections are within per-account page groups, not standalone pages. Must use `split_by_account()` output.
- Multi-line payee entries (e.g., "SHERI MARTIN / CAPITAL ONE N.A. ******0615") span 2-3 lines after the main transaction line. Continuation detection: no MM/DD date prefix.
- Page breaks inject noise into continuation lines: "INVESTMENT REPORT", "Account #", "YI CHEN", single-letter artifacts (`S`), 8+ digit barcode numbers, underscore strings (`BBBBB_FWLBBBBHTDJRB_`), "EC_RM", month names. All must be filtered.
- "Cards, Checking & Bill Payments" in Account Summary is a sub-total category, NOT a separate activity section. The actual items (SAMMPLAT bill pay, checks) appear in Withdrawals and Checking Activity sections respectively.
- Checking Activity uses format `CheckNumber MM/DD Code Description Amount` — different from Deposits/Withdrawals format.
- Exchanges In/Out use securities format `MM/DD SecurityName CUSIP Description Quantity Price Amount` — only need date + description + amount.
- "Insufficient Funds" entries appear as deposits with negative amounts (reversal of failed EFT). Correctly categorized as Transfer In (Reversal) with is_transfer=1.
- Sheri Martin payment classification by amount works well: $1,254 exact = Child Support, $50K exact = Property Settlement, >= $3K = Spousal Maintenance. The few "Divorce — Other" items ($960, $1,895, $2,848) are irregular settlement-related transfers around the May 2025 finalization.
- "Apa treas" = Airbnb rental income (short-term rental payout), "Solium Inc" = RSU/ESPP proceeds (Salesforce equity), "4 Corners Fina" = legal/financial services vendor.
- The finance DB's only irreplaceable data is the 751 categorization rules. All transaction data can be reimported from source CSVs/YAMLs. The rules were built up over many interactive `/finance review` sessions — only ~230 are in the Python seed data.

**What was decided**:
- Auto-export categorization rules JSON on every import (part of `backup_db()`) — no manual step needed
- JSON file syncs via Obsidian Sync, so rules survive even if the DB is deleted or machine is lost
- Full rebuild is now documented as an 8-step procedure in Finance/CLAUDE.md
