## 20:49 — Credit Card Statement Ingestion Pipeline

**What was done**:
- Built end-to-end credit card statement PDF → CSV pipeline
- Created `Scripts/venv/` (recreated from broken Python 3.13 → 3.14)
- Installed pdfplumber 0.11.9
- Explored sample PDFs from each bank to understand text extraction formats
- Implemented 3 parsers: ChaseParser, AppleCardParser, FidelityParser
- Built scaffold with argparse CLI (--scan, --run, --card, --force, --skip-errors)
- Processing log tracks what's been ingested; incremental by default
- Per-statement validation: sums parsed transactions and compares against PDF summary totals
- Created `/ingest-cc` slash command
- Moved script from `Scripts/` to `.claude/scripts/` per user preference (skill files colocated with commands)

**What was learned**:
- Chase PDFs have anti-copy doubled characters in headers (e.g., `AACCCCOOUUNNTT AACCTTIIVVIITTYY`). Transaction text is normal — only section headers affected. Multi-page statements use `ACCOUNT ACTIVITY (CONTINUED)` (NOT doubled) on subsequent pages.
- Chase transaction format: `MM/DD Description Amount` — single date, no post date. Payments are negative, purchases positive. Negate all for CSV convention.
- Apple Card format: `MM/DD/YYYY Description N% $DailyCash $Amount`. Returns lack Daily Cash columns and show as negative amounts. Must try PAY_RE as fallback in transactions section.
- Fidelity format: `MM/DD MM/DD REF# Description $AmountCR?` — two dates (post + trans), use trans date. CR suffix = credit. Single-date entries exist for adjustments, charge-offs, interest — need ALT_RE with optional ref# pattern.
- Chase "Purchases" summary total EXCLUDES fees and interest. Must extract all three and sum for validation. Annual fee for Sapphire Reserve: $550+$75=$625.
- Apple Card "Total charges, credits and returns" is a NET number (charges minus returns), not gross charges.
- Apple Card PDFs sometimes misfiled in wrong year subfolders. Deduplication by filename prevents double-processing. Found `January 2025.pdf` in both `2024/` and `2025/` folders.
- Fidelity Credit Card folder is flat (no year subfolders), unlike all other cards.
- pdfplumber `extract_text()` works well for all three banks. No need for `extract_tables()`.

**What was decided**:
- CSV schema: `date,description,amount,category` — negative=charge, positive=payment/credit
- Statement date derived from filename (never folder path)
- Slash command scripts belong in `.claude/scripts/`, not `Scripts/`
- Validation compares parsed totals against PDF summary; mismatches < $3 accepted as PDF text extraction rounding

**Open threads**:
- Category field is always empty — Apple Card has Daily Cash % but not merchant categories; Chase/Fidelity have none
- Could add merchant categorization as a future enhancement
- Small validation mismatches (~$0.50) on some Chase/Apple statements likely from page-boundary text extraction issues

## 23:28 — Personal Finance Database, Categorization & Analysis

**What was done**:
- Built `/finance` skill: SQLite DB + Python script + slash command
- Schema: transactions, categories, accounts, categorization_rules, import_log
- Imported 4,804 transactions from 140 CSVs across 6 credit cards
- Seeded ~230 keyword categorization rules in script, achieving 82.6% coverage
- Ran 12 rounds of `/finance review` with user confirmation, adding ~270 more rules
- Reached 99.7% categorization (4,634/4,648 spending txns), 14 remaining are charities/credits
- Created `/man` slash command for showing usage docs for other slash commands
- Generated spending visualizations: stacked bar chart PNG + ASCII bar charts in terminal
- Analyzed spending trends: identified Travel, Legal, Shopping as top categories
- Installed matplotlib 3.10.8 in venv for chart generation

**What was learned**:
- Dedup key must be `source_file + source_row`, NOT natural key — same merchant/amount/date happens legitimately (e.g., two $4.80 Starbucks on same day)
- Categorization is a long-tail problem: first 230 rules get 82.6%, need ~500 total for 99.7%
- `/finance review` workflow is effective: show uncategorized grouped by description + count, propose categories, user confirms batch, apply via `add-rule`
- Some keyword rules match way more than expected (e.g., `Kindle Svcs` matched 25, `Letsgo Network` matched 7, `WEIXIN` matched 24)
- User corrections: ROOSTERHAUSRESCUE = kids activity venue, UBER ONE/PASS = transportation not media, SWEET FACTORY = candy store not restaurant, vending machines = fast food not online shopping
- RIVIAN charge was a reservation fee, not car maintenance
- ASCII charts in terminal: user couldn't see Unicode block characters (█▓▒░), switched to simple `#` bars which worked
- User insight: Travel is #1 spend ($32.5K/14mo) but cutting it hurts happiness; Legal ($18K) is gone post-divorce, freeing ~$1,500/month

**What was decided**:
- Finance DB lives at `Finance/finance.db`
- Reports/charts saved to `Finance/reports/`
- Spending queries exclude `is_transfer=1` and use `ABS(amount)`
- Category hierarchy: top-level grouping via `COALESCE(cp.name, c.name)`
- Added Insurance as new top-level category (not in original plan)
- Added Massage under Personal Care (not in original plan)

- Should consider exporting rules back to script for reproducibility
