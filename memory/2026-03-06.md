## 15:03 — Finance Pipeline Minions-Inspired Improvements

**What was done**:
- Implemented 6 improvements to finance pipeline inspired by Stripe's "Minions" article
- Added `cmd_dashboard()` to `finance_db.py` — reads all 6 processing logs, DB stats, detects pending files, reports categorization gaps and freshness warnings
- Added `cmd_preflight()` — checks source dirs, DB writable, venv, pdfminer dependency, processing logs, rules backup
- Added `cmd_rebuild()` — Phase 1: parallel parse (6 subprocess.Popen), Phase 2: sequential import (12 steps). Flags: `--force`, `--import-only`, `--parse-only`
- Added `MAX_PARSE_ATTEMPTS = 3` retry budget to all 6 ingest scripts (cc, tax, payslips, fidelity, sofi, becu)
- Updated `/finance` command: replaced "Setup Check" with "Context Priming" using dashboard; generalized Tax Completeness Guard to Data Completeness Guard
- Updated `Finance/CLAUDE.md`: documented new commands, single-command rebuild, Pipeline Observability table
- Added constants: `PROCESSING_LOGS`, `SOURCE_DIRS`, `INGEST_SCRIPTS`, `SCRIPTS_DIR`, `VENV_PYTHON`

**What was learned**:
- Processing log formats vary wildly across ingest scripts:
  - CC: `cards -> {card_name -> {processed: {filename: ...}}}`
  - Tax: `files -> {filename: {...}}`
  - Payslips: `employers -> {name -> {files_scanned: {filename: ...}, payslips: {...}}}`
  - Fidelity: flat dict `{filename.pdf: {...}}` — no version/last_run wrapper
  - SoFi/BECU: `pdfs -> {filename: {...}}`
- Fidelity's log is the outlier — no `last_run` key, uses `processed_at` per entry
- Pending file detection is tricky: source PDFs may be removed after processing (e.g., BECU has 14 in log but 0 in source dir). Must use log count for "processed", not `total - pending`
- Rules backup format is `{"exported_at": ..., "count": N, "rules": [...]}` — not a flat list
- Background agent pattern works well for independent changes across multiple files (retry budget across 6 scripts completed in ~2 minutes while I worked on finance_db.py)

**What was decided**:
- Processing log format-aware parsing required per-source-name switch logic in `_count_log_files()`
- Preflight uses `_count_log_files()` for "processed" count (not source dir minus pending)
- `cmd_rebuild()` continues on ingest script errors (warns but doesn't abort)
- `cmd_preflight()` returns boolean `ready` to gate `cmd_rebuild()`
