## 18:00 — Tax Document Ingestion Pipeline (Phase 1-3)

**What was done**:
- Built `.claude/scripts/ingest_tax.py` (3,371 lines) — full tax PDF ingestion pipeline
- Phase 1: Script skeleton + W-2 parser (5 format variants) + 1099-R parser + 1098 parser
- Phase 2 (team of 4 agents in parallel): Added 1099-INT, 5498-SA, 5498, 1099-SA, Schedule H parsers
- Phase 3 (team agent): Form 1040 parser for filed returns from archive folder (2022-2024)
- Phase 3 (team agent): Cross-validation between prepare-folder source documents and archive-folder returns
- Created `/ingest-tax` slash command at `.claude/commands/ingest-tax.md`
- Generated 49 YAML files across 2022-2025 (15 W-2s, 9 1099-Rs, 14 1098s, 2 1099-INTs, 3 5498-SAs, 5 5498s, 1 1099-SA, 2 Schedule Hs, 3 1040 summaries)

**What was learned**:
- **W-2 PDF extraction is hard**: Salesforce W-2 has form template (Arial 4.5pt) overlaid with values (Courier 8pt). Must filter by fontname. Multiple copies per page (4 vertical + 2 horizontal). The `_build_tokens()` function needs y-clustering before x-sorting — otherwise chars at slightly different y-positions get interleaved and merged into wrong tokens
- **W-2 "renew-2024" false positive**: Substring "w-2" match catches "renew-2024" because rene**w-2**024. Fixed with word-boundary regex
- **ServiceTitan W-2** (Paylocity format): Values-only layout in Courier_9.0, employer name in Courier_8.5 (different font size group). Must search ALL size groups for employer name
- **ADP W-2s** (Airbnb, Meta): All Helvetica, no font distinction. Used regex on earnings summary section + character-level band extraction
- **1099-R Fidelity 401k layout**: Values on the line AFTER box labels, not inline. Distribution code "G" is 2 lines below the "7 Distribution code(s)" label
- **1040 CPA returns**: 2022 format reversed — amounts appear BEFORE labels (unlike 2023/2024). Each year's return is 35-47 pages
- **Cross-validation findings**: All MFJ years (2022-2024) have expected gaps from spouse data. Backdoor Roth shows $6-7K on 1099-R but $0-3 on 1040 line 4b (Form 8606 basis). Lincoln Life third-party sick pay creates apparent W-2 over-count
- **Team agent pattern works well**: 4 agents editing different functions in same file = minimal conflicts. Use TaskCreate with blockedBy for sequential dependencies. ~15 min wall-clock for 5 tasks that would be ~45 min sequential
- **Historical AGI**: $544,731 (2024 MFJ), $716,907 (2023 MFJ), $664,408 (2022 MFJ)

**What was decided**:
- Phase 4 (SQLite integration) is the next step — load 49 YAMLs into finance DB for `/finance` queries

- Sheri's W-2s detected but not fully parsed (IFS format) — acceptable for separate filing going forward
