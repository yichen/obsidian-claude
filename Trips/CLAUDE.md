## Trip Planning Workflow

### Phase 1: Initial Planning (Cost Comparison)
Start with a planning doc that includes:
- Travel dates (arrival/departure)
- **Check Parenting Schedule**: Confirm custody dates in [Children/parenting-schedule-2-2-5-5.md](../Children/parenting-schedule-2-2-5-5.md)
- Estimated total cost
- Hotel/package options with ratings and costs
- Transportation options (car rental, flights if not packaged)
- **Keep this section at the top** - helps with decision-making

**Example format:**
```markdown
# [Destination] Trip Planning

High-Level Plan:
- Leave on [Day/Date], Return on [Day/Date]
- **Check Parenting Schedule**: Confirm custody dates in [Children/parenting-schedule-2-2-5-5.md](../Children/parenting-schedule-2-2-5-5.md)
- Estimated Cost: $X,XXX - $Y,YYY

## Accommodation Options (Costco/other packages)
- Hotel Name 1
  - Rating: X/5
  - Cost: $X,XXX (includes flight/hotel/car)
  - Location notes
- Hotel Name 2
  - Rating: X/5
  - Cost: $X,XXX
```

### Phase 2: Itinerary Development
After choosing accommodation, append (don't replace) detailed itinerary:

**Required elements for each day:**
1. **Driving distances and times FROM THE HOTEL** for every destination
2. **URLs with photos/descriptions** for each point of interest (prefer AllTrails, NPS, etc.). Also find a popular YouTube video about the destination (prefer recent videos from the last 5 years with high popularity). **⚠️ All URLs MUST be validated using WebFetch before including them - see "Link Validation" rule in Agent Rules section.**
3. **Estimated activity durations**
4. **Total daily driving time**
5. **Meal/break suggestions**

**Example format:**
```markdown
## Day X - [Day of week, Date]: [Theme]

**Morning:**
- Activity name
- **Drive from hotel: X hours X minutes (X miles)** via [Route]
- Description
- [URL with photos]

**Afternoon:**
- Next activity
- **Drive from previous location: X minutes (X miles)**
- **Total from hotel: X hours X minutes (X miles)**

**Total driving: ~X hours**
```

### Phase 3: Supporting Sections
Include at end:
- **Driving summary table** (by day)
- **Alternative options** (backup hikes, bad weather plans)
- **Packing checklist** (specific to trip type)
- **Logistics & tips** (car rental notes, weather, food strategy)
- **Key distances reference** (quick lookup from hotel)
- **Emergency contacts**

## Usage Guidelines

### For AI Analysis
When assisting with travel planning or reviewing past trips:

1. **Identify trip type first:** Solo vs. Family
   - Solo: Focus on challenging hikes, exploration, minimal beach time, include work buffer time
   - Family: Kid-friendly activities, moderate hikes, beach time, educational stops

2. **Check year-based folders** for relevant past trip reports
   - Look for "Lessons Learned" sections
   - Note what worked well vs. what to avoid

3. **Reference similar trips** to inform new planning
   - Solo trips: Banff (hiking-focused, scenic drives)
   - Family trips: Kauai (beach/water activities), Olympic NP (camping/hiking)

4. **Start with cost comparison** if user provides hotel options
   - Recommend best basecamp based on activity access
   - Consider driving times to major attractions

5. **Build detailed itineraries** with:
   - All driving times/distances from hotel
   - **Validated URLs** with photos and a popular YouTube video for every POI (use WebFetch to verify each link works)
   - Daily driving totals
   - Work time buffers for solo weekday trips

6. **Consider seasonality** based on trip dates
   - Weather impacts on hiking/driving
   - Crowd levels at popular destinations

7. **Extract lessons learned** from past trips
   - Packing insights (Banff: brought too many clothes, needed rain pants)
   - Activity timing (start hikes early to avoid heat)
   - Logistics (roof box parking issues, gas station planning)

## Trip Lifecycle & `/trip` Command

Use `/trip` to manage the full trip lifecycle. It replaces ad-hoc file creation and keeps every phase organized.

### Folder Structure

```
Trips/PLAN-FUTURE/Trip-Name/        ← staging while exploring options
Trips/YYYY/YYYY-MM-DD_Trip-Name/    ← committed trips

  README.md         status + quick-ref (confirmation numbers)
  Research.md       destination options, cost comparisons
  Itinerary.md      day-by-day plan
  Reservations.md   booking confirmations
  journal/
    YYYY-MM-DD.md   daily notes written during the trip
  Report.md         post-trip highlights, lessons, actual costs
```

Simple camping weekends with no bookings: single `.md` file is fine.

### Commands

| Command | Phase | What it does |
|---------|-------|-------------|
| `/trip new "Name"` | Explore | Creates folder in `PLAN-FUTURE/` with README + Research stub |
| `/trip research` | Explore | Researches/compares destinations, writes to Research.md |
| `/trip commit` | Plan | Moves folder to `Trips/YYYY/`, creates Itinerary stub, status → PLANNED |
| `/trip book` | Book | Updates Reservations.md with confirmations, status → BOOKED |
| `/trip pack` | Pre-trip | Generates packing list from Lessons Learned + trip type |
| `/trip log [note]` | During | Auto-detects active trip, appends to `journal/YYYY-MM-DD.md` |
| `/trip report` | Post-trip | Fills Report.md, pushes lessons to Lessons Learned.md, updates Trip Index |
| `/trip <question>` | Any | Natural language search across all past trips |

### Status Lifecycle
`EXPLORING → PLANNED → BOOKED → IN PROGRESS → COMPLETED`

### 6. Stress-Aware Trip Recommendations
**When the user asks about trip planning, proactively check for elevated stress levels.**

**Detection:**
1. Read recent journal entries from `Journal/Daily/` (last 3-5 days)
2. Look for indicators of elevated stress:
   - Explicit mentions of stress, anxiety, or poor sleep
   - High stress levels from watch data (e.g., stress level > 50)
   - Multiple consecutive days of work pressure or difficult situations

**If elevated stress is detected:**
1. Acknowledge the observation (e.g., "I noticed from recent journal entries that you've been under significant stress...")
2. Suggest destinations with heated pools/hot tubs from the "Stress Relief & Wellness Travel" section above
3. Emphasize the proven de-stressing benefits (reference: Suncadia trip where stress dropped from 80 to 28)
4. For quick relief, prioritize closer destinations (Suncadia ~1.5hr, Alderbrook ~2hr)

### 7. Weather Forecasting (MANDATORY)
**ALWAYS use forecast.weather.gov for weather forecasts. NEVER use web search for weather data.**

**Process:**
1. **Find the NWS forecast URL** for the destination:
   - Go to https://forecast.weather.gov
   - Search for the city/location or use lat/lon coordinates
   - Get the specific forecast URL (format: `https://forecast.weather.gov/MapClick.php?lat=XX.XXX&lon=-XXX.XXX`)

2. **Use WebFetch to retrieve the official forecast:**
   - Call WebFetch with the NWS URL
   - Request detailed daily forecasts including temps, precipitation %, conditions
   - Parse the official NWS data only

3. **NEVER use WebSearch for weather:**
   - Web search results are unreliable, may be cached, or show wrong locations
   - Third-party weather sites may have outdated or inaccurate data
   - Only use official NWS data from forecast.weather.gov

**Example:**
```
WebFetch(
  url: "https://forecast.weather.gov/MapClick.php?lat=47.195&lon=-120.9394",
  prompt: "What is the detailed weather forecast for February 6-8, 2026? Include high/low temps, precipitation chances, and conditions for each day."
)
```

### 8. Link Validation (MANDATORY)
**All URLs included in trip plans MUST be validated before adding them.** Broken links waste time and create frustration during trips.

**Validation Process:**
1. **Use WebFetch to verify each URL** returns a valid page (not 404, 403, 410, or connection errors)
2. **Confirm the content matches** the expected destination (e.g., a restaurant link should show restaurant info, not a generic homepage)
3. **Never use placeholder URLs** like `example1`, `placeholder`, or made-up paths
4. **Prefer official sources** in this order:
   - Official website (e.g., `nps.gov`, `parks.ca.gov`, restaurant's own site)
   - Established guides (e.g., `alltrails.com`, `visitXXX.com` tourism sites)
   - Review sites (e.g., `tripadvisor.com`, `yelp.com`) as fallback
5. **For YouTube videos:** Use YouTube search result links (`youtube.com/results?search_query=...`) rather than specific video IDs that may become unavailable

**Common URL patterns that break:**
- Restaurant websites often change structure (verify before adding)
- HikingGuy, ModernHiker URLs sometimes return 403/410
- Tourism sites reorganize their `/places/` or `/directory/` paths
- Event-specific URLs (like dance socials) expire after the event date

**If a URL cannot be validated:**
- Search for an alternative working URL for the same destination
- If no working URL exists, note the destination name without a link and add "(verify URL before trip)"

**Trail Status Check:** For hiking trails, also verify the trail is OPEN (check for fire closures, seasonal closures, permit requirements) and note any closures prominently.
